# Thogakade Database CRUD - JavaFX

Complete CRUD operations for Thogakade retail management system covering Customers, Items, Orders, and Order Details using JavaFX. This project follows the MVC (Model-View-Controller) architecture pattern and implements the Singleton Design Pattern for database connections.

## 📋 Project Overview

This repository demonstrates comprehensive database CRUD (Create, Read, Update, Delete) operations for a retail management system called "Thogakade". Built with JavaFX, it provides a clean, professional interface for managing all aspects of a retail business including customer information, inventory items, and order processing with detailed order entries.

## 🏗️ Architecture

- MVC (Model-View-Controller) Pattern
- Model: Entity classes representing database tables

- View: FXML files for user interface

- Controller: Business logic and event handling

- Layered Architecture

## Design Patterns Implemented

- Singleton Pattern: Single database connection instance

- DAO Pattern: Data access objects for database operations

- Factory Pattern: Object creation (optional implementation)

## 🛠️ Technologies Used

- Java - Core programming language (JDK 8+)

- JavaFX - Rich client application framework

- Scene Builder - FXML UI design tool

- FXML - XML-based UI markup

- CSS - Application styling

- MySQL - Relational database management

- JDBC - Database connectivity

- Maven/Gradle - Build automation

## 🔍 Key Learning Outcomes

- Implementing MVC architecture in JavaFX

- Using Singleton pattern for database connections

- Performing CRUD operations with JDBC

- Handling complex transactions (Orders + OrderDetails)

- Managing foreign key relationships

- Validating user inputs

- UI design with FXML and CSS
